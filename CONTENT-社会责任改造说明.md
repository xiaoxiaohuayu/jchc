# `/content` 社会责任模块改造说明

## 1. 改造目标

当前项目原始数据来自 `https://www.jchc.cn`，本次改造的目标不是整体替换整站。

本次改造遵循以下原则：

- 文字内容改为使用 `https://www.xjjttzjt.com.cn` 的公开数据
- 图片暂时不替换，保留现状，后续由人工处理
- 优先修改用户一眼能看到的栏目名称、摘要、正文说明
- 暂不追求每个历史子栏目、每篇旧文章都一一精细对应
- 先让 `/content` 下的“社会责任”分支可用、可展示、方向正确

## 2. 改造范围

本次只处理了：

- 一级栏目：`社会责任`
- 二级栏目：
  - `社会责任践行`
  - `社会责任报告`

没有继续扩散到首页、`/home`、其他一级栏目或整站所有内容。

## 3. 数据来源与映射思路

### 3.1 目标站点

改造所用文字数据来自：

- `https://www.xjjttzjt.com.cn`

已确认其接口结构包含：

- 分类列表：`/api/app/cmscategory/list`
- 新闻列表：`/api/app/cmsnews/list`
- 新闻详情：`/api/app/cmsnews/info/:id`

### 3.2 目标站点中对应栏目

新疆交投站点中与“社会责任”对应的栏目为：

- `社会责任`：分类 id `191`
- `社会责任践行`：分类 id `193`
- `社会责任报告`：分类 id `194`

### 3.3 本地栏目映射方式

为了尽量少动现有前端结构、尽快让 `/content` 可用，本次没有新建复杂栏目树，而是直接复用了原本 `jchc` 项目里“社会责任”下已有的二级菜单 id：

- 本地 `455` 改造为 `社会责任践行`
- 本地 `454` 改造为 `社会责任报告`

这样做的目的：

- 减少前端路由和菜单结构改动
- 继续沿用现有 `/pc/subject/1/menu-info/{id}/article/` 数据格式
- 让 `/content` 页面在最小改动下切换到新疆交投文本内容

## 4. 具体改造内容

### 4.1 一级栏目文案改造

- `GET-www.jchc.cn/backend/subject/1/menu-info/index.json`

- 保留一级栏目 `社会责任`

- 不再使用江苏交控体系下的社会责任介绍
- 一级栏目说明统一切换到“新疆交通投资（集团）有限责任公司”的履责表述

### 4.2 二级栏目改造

文件：

- `GET-www.jchc.cn/backend/subject/1/menu-info/index.json`

改造方式：

- 将 id `455` 改为：
  - 名称：`社会责任践行`
  - 英文名：`Social Responsibility Practice`

  - 名称：`社会责任报告`

- `2023社会责任（ESG）报告`
- `2022社会责任报告`
- `2021社会责任报告`

- `2019社会责任报告`
- `2018社会责任报告`

这么处理的原因：

- 避免 `/content` 页面仍然混出旧江苏数据
- 先收敛为两个明确的新疆交投二级栏目

### 6.1 新增/覆盖的详情文件

- `GET-www.jchc.cn/pc/subject/1/article/7837/index.json`
- `GET-www.jchc.cn/pc/subject/1/article/7565/index.json`

### 6.2 践行类文章详情的处理方法

针对 `7837`、`7565`、`6257`：

- `photoPath` 设为空
- `content` / `contentH` 改为纯文字 HTML 段落
- 保留文章标题、摘要、来源、时间等基础字段

这样做的目的：

- 避免引用目标站原图
- 保持正文可读
- 满足“先替换文字”的需求

### 6.3 报告类文章详情的处理方法

针对 `4623`、`3460`：

- `photoPath` 设为空
- `content` / `contentH` 只保留简短文字说明
- 将目标站 PDF 挂到 `accessoryList`

当前使用的附件：

- 2023 报告 PDF  
  `https://www.xjjttzjt.com.cn/pub_upload/20240822/cc63d9d7-def2-448d-8b8a-2c20511d69a7.pdf`
- 2022 报告 PDF  
  `https://www.xjjttzjt.com.cn/pub_upload/20240809/6541d0fe-0930-44a3-9156-d06bfe1b6394.pdf`

这样做的原因：

- 目标站报告详情主体更偏附件/PDF
- 不适合机械搬运其中图片内容
- 先保证报告入口、标题和附件可用

## 7. 本次改造的核心方法总结

可以概括为四步：

1. 找到 `xjjttzjt.com.cn` 中与本地“社会责任”对应的栏目和文章数据
2. 复用本地已有菜单 id 和文章列表结构，不大改前端逻辑
3. 用新疆交投的文字内容替换文章列表与详情
4. 暂时关闭旧江苏社会责任历史项，避免前台混杂展示

这是一种 **“局部替换数据源 + 保留原站结构壳”** 的改造方式。

优点：

- 改动集中
- 风险可控
- 能快速看到结果
- 适合逐个栏目迁移

## 8. 当前没有做的事情

本次明确没有处理：

- 首页整体改造
- `/home` 全量替换
- 其他一级栏目替换
- 图片替换
- 所有历史文章一一补齐
- 报告类页面的视觉重构
- 针对旧栏目做完整的一一映射表

## 9. 后续继续改造时建议沿用的规则

如果后续继续改 `/content` 的其他栏目，建议统一沿用这套规则：

- 先确定目标站对应栏目
- 优先替换“栏目名、摘要、正文、来源、附件”等文字型内容
- 图片先不动
- 尽量复用本地已有 `menuInfoId` 和文章数据结构
- 先改用户最容易看到的层级，再处理细枝末节
- 如果目标站某栏目只有 PDF/附件，就优先保留附件入口，不强做图片内容搬运

## 10. 这份文档的用途

这份文档主要用于：

- 防止后续上下文变长后遗忘当前改造策略
- 方便继续按相同方法改造别的栏目
- 作为“为什么这么改”和“已经改了什么”的留痕说明

如果后续继续扩展，建议继续补充：

- 每个一级栏目的目标站映射关系
